# 现代C++17/20 常用语法

## 1. STL预备知识
### 1.1 STL 三件套
```txt
容器:存储数据
算法：处理数据
迭代器：把容器的元素暴露给算法的通用接口
```
#### 关键用法
大多数写法：
```
算法(begin,end,其他参数...)
```
> begin指向第一个元素，end指向最后一个元素**后面**的位置。
### 1.2 区间规则
通常是半开半闭区间`[begin,end)`，也就是包含`begin`指向的元素但是不包含`end`指向的位置。
示例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>x = {3,1,2};
    std::sort(x.begin(),x.end());
    for(int i:x) std::cout << i << std::endl;
    return 0;
}
```
结果：
```
1
2
3
```
- 注：`end()`不是最后元素，不能被解引用。
## 2.C++17常用语法糖
### 2.0 语法糖的解释
**语法糖**是编程语言提供的一种便捷语法—— 它**不增加**语言的核心功能，只是对已有语法的 **“包装”** 或 **“简化”**，让代码写起来更简洁、读起来更直观，但其底层实现和原本的繁琐写法完全一致。
简单来说：**语法糖** = **简化写法** + **等价的底层逻辑**，核心是 “甜化” 代码，降低编写和理解成本，不改变语言的本质能力。
### 2.1 auto
把类型推导交给编译器，使人专注变量语义而不是拼写类型。
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    auto x = 42;
    auto name = std::string("Tom");
    auto nums = std::vector<int>{3,1,2};
    std::cout << x <<" " << name << " " << nums.size();
    return 0;
}
```
输出结果
```
42 Tom 3
```
- 注：右侧表达式易懂时应用`auto`，接口参数和关键变量的类型不明显时，适当显式写类型。
### 2.2 结构化绑定
1. 概念：结构化绑定可以把`pair`,`tuple`或符合要求的结构体**一次性拆解成多个独立变量**
2. 适用范围：
    1. `std::pair`
    2. `std::tuple`
3. `std::pair`简介：
    1. 本质：【二元组】 把两个任意值打包成一个整体
    2. 固定长度(两个元素)，两元素无`键值`关系，吴查找功能
4. 例
```C++
#include <iostream>
#include <algorithm>
#include <map>
#include <utility>
int main(){
    std::pair<int,std::string> p{1,"Alice"};
    auto[id,user] = p;
    std::cout << id << " " << user << std::endl;
    std::map<std::string,int> score{{"math",99},{"cs",95}};
    for(const auto &[k,v]:score)
    {
        std::cout << k << ": " << v << std::endl;
    }
    return 0;
}
```
结果
```
1 Alice
cs: 95
math: 99
```
- 注：遍历通常写`const auto &[k,v]`,而非`auto [k,v]`(避免复制元素，过慢)
### 2.3 Lambda
1. 背景：只需要传一个小逻辑的算法
2. `Lambda`是匿名函数对象，常见形式
```
[捕获] (参数) -> 返回类型{函数体}
```
返回类型大多数可以省略。
3. 捕获的直觉
- 值捕获`[x]`：复制一份到`Lambda`里。
- 引用捕获`[&x]`:Lambda里操作的是外部变量本体。
4. 例：
```C++
#include <algorithm>
#include <iostream>
#include <vector>
int main() {
    std::vector<int> a{1,2,3,4,5};
    int bias = 10;
    std::for_each(a.begin(),a.end(),[bias](int &x){x += bias;});
    int even = std::count_if(a.begin(),a.end(),[](int x){return x % 2 == 0;});
    for(int x : a) std::cout << x << " ";
    std::cout << std::endl << "even = " << even << std::endl;
}
```
5. 注：
- `for_each`里形参是`int &x`，会修改容器元素
- `count_if`的Lambda返回布尔值，满足条件就计数
- 二者的一般形式
```
for_each{
    起始迭代器，
    结束迭代器，
    一元操作函数/Lambda
}
```
### 2.4 if与switch初始化语句
1. C++17允许在`if` 和`switch`中先写一个初始化语句，再写条件判断。
目的：希望某个临时变量只在这一段分支逻辑里，不泄露到外层，不和外层变量重名。
2. 用法：
```
if (init; cond) { ... } else { ... }
switch (init; cond) { ... }
```
其中：
- `init`会先执行一次，常见是声明一个变量。
- `cond`里可以使用`init`声明的变量。
3. 例：
```C++
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string,int> mp{{"bob",95},{"alice",90}};
    if(auto it = mp.find("alice");it != mp.end())
    {
        std::cout << it->first << ": " << it->second << "\n";
    }
    else std::cout << "not found";
    // 这里访问不到 it, 因为 it 的作用域被限制在 if 语句里
}
```
```C++
#include <iostream>
int classify(int x)
{
    if(x < 0) return -1;
    if(x == 0) return 0;
    return 1;
}
int main(){
    switch(int c = classify(7);c)
    {
        case -1:
        std::cout << "negative" << std::endl;
        case 0:
        std::cout << "zero" << std::endl;
        case 1:
        std::cout << "positive" << std::endl;
        break;
    }
    return 0;  
}
```
### 2.5 属性与编译器提示
1. 属性写法`[[....]]`不改变语义，但是会让编译器在高风险代码时给出明确提示。
2. 
    1. `[[nodiscard]]`
    希望调用方不要忽略返回值
    例：
    ```C++
    #include <iostream>

    [[nodiscard]] int compute() { return 42; }

    int main() {
        compute(); // 很多编译器会对忽略返回值发出警告
        int x = compute();
        std::cout << x << "\n";
    }
    ```
    会报错
    2. `[[maybe_unused]]`
    确实需要一个变量或参数，但是某些时候用不上
    ```C++
    #include <iostream>

    void log_value([[maybe_unused]] int x) {
    // 调试时可能打印, 线上关闭日志后 x 就会变成未使用
    std::cout << "log enabled\n";
    }
    ```
    3. `[[fallthrough]]`
    明确想要从`switch`的一个`case`继续执行到下一个`case`时
    ```C++
    #include <iostream>

    int main() {
        int level = 1;
        switch (level) {
        case 1:
        std::cout << "basic\n";
        [[fallthrough]];
        case 2:
            std::cout << "advanced\n";
            break;
        default:
            break;
        }
    }
    ```
## 3.STL常用算法
### 3.1 std::for_each
对区间内每个元素执行一次操作

用法 
```C++
std::for_each(first,last,func)
``` 
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>v{1,2,3};
    std::for_each(v.begin(),v.end(),[](int &x){x *= 10;});
    for(int x:v) std::cout << x << " ";
    
}
```
结果
`10 20 30`
### 3.2 std::count_if
统计区间内满足条件的元素个数。它不会修改元素。

用法：
```C++
std::count_if(first,last,pred)
```
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>v{1,2,3,4,5,6};

    int even = std::count_if(v.begin(),v.end(),[](int x) {return x % 2 == 0;});
    std::cout << even;
}
```
结果
`3`

注：谓词`pred`尽量写成纯判断函数, 不要在里面修改外部状态, 否则读代码的人很难确认你是不是在靠副作用实现逻辑。
### 3.3 std::find_if
在区间里找第一个满足条件的元素, 返回**迭代器**。找不到时返回`end`。

用法：`auto it = std::find_if(first,last,pred)`
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
   std::vector<int>v{3,7,11,20};
   auto it = std::find_if(v.begin(),v.end(),[](int x){return x > 10;});
   if(it != v.end()) std::cout << "found: " << *it <<"\n";
   else std::cout << "not found";
}
```
结果
`found: 11`
### 3.4 std::transform
把输入区间的每个元素变换成输出区间的元素。它相当于把一个循环里的赋值写成了算法。

用法：

常见的一元变换写法：
`std::transform(first,last,out,op)`

其中`out`为任意类型的**输出迭代器**
例：计算平方并放入数组
```C++
#include <iostream>
#include <algorithm>
#include <vector>
#include <iterator>
int main(){
   std::vector<int> v{1,2,3,4};
   std::vector<int> out;
   out.reserve(v.size());
   std::transform(v.begin(),v.end(),std::back_inserter(out),[](int x){return x *x;});
   for(int x:out) std::cout << x << " ";
}
```
结果：`1 4 9 16`
- 注`std::back_inserter(out)`会把赋值操作转成 push_back, 因此你不需要提前 `resize` 输出容器。你仍然最好 `reserve` 一下, 避免频繁扩容。
### 3.5 std::accumulate
把一段区间累积成一个值, 常用于求和, 求积, 或把多个值合成一个结果。

用法：
```C++
std::accumulate(first,last,init)
std::accumulate(first,last,init,op)
```
它在内部按顺序做`init = op(init, x)`。
例：
```C++
#include <iostream>
#include <numeric>
#include <vector>
int main(){
   std::vector<int> v{1,2,3,4};
   int s = std::accumulate(v.begin(),v.end(),0);
   std::cout << s << "\n";
}
```
结果：`10`。

`init` 的意义

`init` 不只是初始值, 它还会影响结果类型。例如你希望在整数求和时用更大的类型防止溢出, 可以写 `0LL`:
```C++
long long s = std::accumulate(v.begin(), v.end(), 0LL);
```
### std::sort与二分查找
排序与二分查找经常成套出现: 先排序, 再用二分在对数时间内查找

std::sort
```C++
std::sort(begin, end)
std::sort(begin, end, comp)
```
`comp(a,b)`返回true表示a应该排在b前面。

std::lower_bound

`lower_bound`在已排序区间内找第一个不小于目标值的**位置**。
```C++
auto it = std::lower_bound(begin,end,value)
```
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int> v{5,1,4,2,3};
    std::sort(v.begin(),v.end());
    int target = 4;
    auto it = std::lower_bound(v.begin(),v.end(),target);
    if(it != v.end() and *it == target) std::cout << "found " << target << "\n";
}
```
结果：`found 4`
- 注：二分查找类算法要求区间已经按同样的规则排好序, 否则结果不可信。
### std::remove_if 与 erase-remove
`remove_if`在很多容器上不会真正缩短长度

对`vector`这类容器，`std::remove_if`会把不需要删除的元素移到前面，返回一个新的逻辑结尾迭代器。用`erase`删除。

用法：
`v.erase(std::remove_if(v.begin(),v.end(),pred),v.end());`
例：删除所有奇数
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int> v{1,2,3,4,5,6};
    v.erase(std::remove_if(v.begin(),v.end(),[](int x){x % 2 == 1};),v.end());
    for(int x : v) std::cout << x << " "; 
    return 0;
}
```
运行结果：

`2 4 6`

解读：
`remove_if`负责把“要保留的元素”挪到前面，`erase`负责删除尾部多余元素

`remove_if`讲解及示例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>v {1,2,3,4,5,6};
    auto m = std::remove_if(v.begin(),v.end(),[](int x) { return x % 2 == 1;});
    for(std::vector<int>::iterator it = v.begin();it!=v.end();++it) std::cout << *it << " ";

}
```
运行结果：

`2 4 6 4 5 6`

`remove_if`把需要保留的元素挪到前面后，后几个位置的元素不变(包括值和顺序，容器中元素和原来可不一样)，而不是简单调换位置。
## 4. 值类别与移动语义
**本章是现代C++的核心。**

### 4.1 函数参数传递方式
#### 概念
- 值传递`T`:传进去是一份副本
- 可写引用`T&`:允许函数修改调用方对象
- 只读引用`const T&`:不复制，只读，也能接临时对象
- 右值引用`T&&`:常用于移动语义或转发

注： 
1. 接临时对象：C++规定非常量引用(T&)不能绑定到临时对象。   
    原因：临时对象马上就要销毁，使用非常量引用可能会通过引用修改，导致未定义行为。
2. 右值引用：`T&&`是专门绑定临时对象(右值)的引用

    1. 移动语义——省内存 ：因为临时对象马上就要销毁，可以直接使用其内存，而不需要复制
    例：
    ```C++
    #include <iostream>
    #include <string> 
    // 假设这里有一个大字符串拼接操作
    std::string getLongString() {
        return "This is a very long long long string...";
    }
    int main() {
        // 1. 传统方式：发生拷贝
        // std::string s = getLongString(); 
        // 流程：生成临时对象 -> 拷贝给 s -> 销毁临时对象
        // 2. 使用移动语义（T&&）
        std::string&& s = getLongString(); 
        // 流程：直接把临时对象的“资源指针”移交给 s，不发生拷贝，效率极高
        return 0;
    }
    ```
### 4.2 左值，右值，引用
#### 判别方法：
- 有名字，能反复用的变量表达式，大多数是左值。
- 字面量，临时对象，表达式计算出来的临时结果，大多数是右值。
#### 术语补充
- `lvalue`：左值表达式，例如变量名`x`。
- `prvalue`:纯右值，例如字面量`42`或临时对象`std::string("tmp")`。
- `xvalue`:将亡值，例如`std::move(x)`的结果。它依然引用着某个对象，但通常表示该对象的资源可以被接管。
例：
```C++
#include <iostream>
#include <utility>
void f(int&) {std::cout << "f(int&)\n";}
void f(const int &){std::cout << "f(const int&)\n";}
void f(int &&){std::cout << "f(int&&)\n";}
int main(){
    int x = 10;
    const int cs = 20;

    f(x);
    f(cs);
    f(30);
    f(std::move(x));
}
```
运行结果：
```
f(int&)
f(const int&)
f(int&&)
f(int&&)
```
注：这段代码展示的是重载选择，不是资源真的被搬走
### 4.3 std::move
#### 概念
`std::move`的作用是把表达式转换成右值，更准确地说是`xvalue`。它本身不搬资源，它知识告诉编译器把这个对象当作可被移动的来源，让重载决议更倾向于选择移动构造或移动赋值。
示例：
```C++
#include <iostream>
#include <string>
#include <utility>
int main(){
    std::string a = "hello";
    std::move(a);
    std::cout << a << std::endl;
    std::string b = a;
    std::cout << "b = " << b << " " << "a.size() = "<< a.size() << std::endl;
    b = std::move(a);
    std::cout << "b= " << b << "\n";
    std::cout <<"a.size()= " << a.size() << "\n";
    std::cout << a << std::endl;
    std::cout << &a;
}
```
输出结果：
```
hello
b = hello a.size() = 5
b= hello
a.size()= 0

0xfbd49ff710
```
解释：
`std::move`只是一次类型转换，是后续发生的移动构造或移动赋值以及这些移动操作内部对源对象的处理方式让对象进入`moved-from`状态，拷贝构造和拷贝赋值不会让对象进入`moved-from`状态，单独写一行`std::move(x)`**不会触发任何资源转移**。
#### return时不要写std::move
通常不要写。写`return local`更好
原因：
- 编译器可能做返回值优化，直接在返回位置构造对象。
- 即使不做优化，编译器也可以把局部变量当作可移动对象处理。

例：
```C++
#include <string>
std::string make_name() {
    std::string s = "Tom";
    return s; // 通常写法, 不要强行 std::move(s)
}
```
### 4.4 moved-from对象还能不能用
#### 结论
能用，但只能但一个仍然可析构，可重新赋值的对象
#### 推荐做法
- `move`之后如果还要继续使用，应重新赋值
### 4.5 转发引用与引用折叠
#### 概念
只有当形参写成`T&&`，且`T`需要由实参推导得到时(在模板函数且参数声明为`T&&`且`T`为模板参数)才成立。它的特点时既能绑定左值也能绑定右值，并把值类别的信息编码进`T`的推导结果里。

若修改写法，就不再是转发引用：
- `const T&&`不是转发引用，只能绑定右值
- `std::vector<T>&&`不是转发引用，它只是某个具体类型的右值引用，也只能绑定右值

同样的规则也适用于`auto&&`，例如泛型Lambda的形参或range-for中的`auto&& x`。

引用折叠规则：
| 折叠组合                 | 结果                  | 记忆口诀       |
|--------------------------|-----------------------|----------------|
| 左值引用 + 左值引用      | `T& &` → `T&`         | 有左值则左值   |
| 左值引用 + 右值引用      | `T& &&` → `T&`        | 有左值则左值   |
| 右值引用 + 左值引用      | `T&& &` → `T&`        | 有左值则左值   |
| 右值引用 + 右值引用      | `T&& &&` → `T&&`      | 全右值才右值   |
#### f(T x)按值接收时的推导
当形参是`T x`，`T`的推导更像是做一次按值拷贝需要的类型标准化。
- 实参如果是引用，会先去掉引用再推导`T`
- 实参如果带顶层`const`,推导`T`会去掉这个const
- 数组和函数类型会退化成指针类型

结果是`f(T x)`几乎不可能保留调用方传入的是左值还是右值，也不可能保留顶层`const`。这也是为什么完美转发不可能靠按值参数实现。
####  类型和表达式值类别是两件事
**右值引用不等于右值表达式**
当你在函数体里写`x`时，`x`是一个变量名，它作为表达式属于左值。如果希望把它当作右值去选择右值重载或出发移动语义，需要显示写`std::move(x)`或`std::forward<T>(x)`。
例：
```C++
#include <iostream>
#include <type_traits>

template <class T>
void show_forward(T&&) {
    std::cout << std::boolalpha
              << "forward: T_is_lref=" << std::is_lvalue_reference_v<T>
              << ", T_base_const=" << std::is_const_v<std::remove_reference_t<T>> << "\n";
}

template <class T>
void show_value(T) {
    std::cout << std::boolalpha
              << "value:   T_is_lref=" << std::is_lvalue_reference_v<T>
              << ", T_base_const=" << std::is_const_v<std::remove_reference_t<T>> << "\n";
}

int main() {
    int x = 1;
    const int cx = 2;

    show_forward(x);
    show_value(x);

    show_forward(cx);
    show_value(cx);

    show_forward(3);
    show_value(3);
}
```
输出：
```
forward: T_is_lref=true, T_base_const=false
value:   T_is_lref=false, T_base_const=false
forward: T_is_lref=true, T_base_const=true
value:   T_is_lref=false, T_base_const=false
forward: T_is_lref=false, T_base_const=false
value:   T_is_lref=false, T_base_const=false
```
注：`T&&`是一种省略参数的写法，是合法的
### std::forward
<span style="color: #FF0000; font-size: 18px;">此处很重要</span>

一句话总结`std::forward`的目标：让下游函数看到的值类别尽量等于调用方传进来的值类别。

注：值类别是表达式的属性，不是类型的属性。即使一个变量的类型是右值引用，它只要有名字，这个名字再表达式规则里就是左值。

例1：
```C++
#include <iostream>
#include <type_traits>

void sink(int &) {std::cout << "sink(int&)\n";}
void sink(int &&) {std::cout << "sink(int &&)\n";}

int main() {
    int && rr = 3;  //rr 的类型是 int &&
    sink(rr);       //rr 这个名字是左值表达式
    sink(std::move(rr));    //move把表达式转换成右值
}
```
解释：`rr` 的类型是`int &&`,但`rr`作为表达式仍然走左值重载，这不是模板特例，而是 C++ 表达式规则。

模板里的形参`x`也是同样道理：用`T&&`借助==接住一个右值后，`x`的声明类型可能是`U&&`，但在函数体里直接写`x`时，它依然是左值表达式，所以需要`forward`才能恢复成合适的值类别。
#### 第一步 先观察坏掉的地方
下面这段代码里，调用方传了右值`3`,但`sink(x)`仍然走左值重载。原因是`x`是一个有名字的变量表达式。
```C++
#include <iostream>
void sink(int&) { std::cout << "sink(int&)\n"; }
void sink(int&&) { std::cout << "sink(int&&)\n"; }
template <typename T>
void wrapper_bad(T&& x) {
    sink(x); // x 有名字, 表达式永远是左值
}

int main() {
    int a = 1;
    wrapper_bad(a);
    wrapper_bad(3);
}
```
#### 第二步 T 是怎么推导出来的
现在看`wrapper_bad(T && x)`这一行。这里的`T&&`处于模板推导语境，所以是转发引用，推导规则为：
- 如果实参是左值`U`，那么`T`推导为`U&`
- 如果实参是右值`U`，那么`T`推导为`U`

加上引用折叠，结果如下：
|调用方式            |T 的推导结果        |形参类型 T&& 的折叠结果|
|-------------------|--------------------|-------------|
|wrapper_bad(a)     |int&               |int&|
|wrapper_bad(ca)     |const int&         |const int&
|wrapper_bad(3)      |int                |int&&

注：这张表只是说类型，并未说表达式。即使`x`的类型是`int&&`，在函数体里写`x`时，依然是左值表达式。
#### 第三步 forward的语义
`std::forward<T>(x)`的本质是一次条件转换：
- 当`T`是左值引用类型时，它把`x`当作左值返回
- 当`T`不是左值引用类型时，它把`x`当作右值返回

可以理解为：
>`static_cast<T&&>(x)`

这行`cast`能起作用的关键还是引用折叠
|               |                 |                                                           |
|---------------|-----------------|---------|
|如果 T = U&    |那么 T&& 折叠成 U&|     static_cast<T&&>(x) 等价于 static_cast<U&>(x)|
|如果 T = U     |那么 T&& 就是 U&& |    static_cast<T&&>(x) 等价于 static_cast<U&&>(x)|

因此`forward<T>(x)`利用`T`的推导结果把`x`转回合适的值类别

#### 第四步 正确写法与运行结果
```C++
#include <iostream>
#include <type_traits>

void sink(int &) {std::cout << "sink(int&)\n";}
void sink(const int&) {std::cout << "sink(cons int&)\n";}
void sink(int &&) {std::cout << "sink(int &&)\n";}

template<typename T>
void wrapper_bad(T && x){sink(std::forward<T>(x));}
int main() {
    int a = 1;
    const int ca = 2;  

    wrapper_bad(a);
    wrapper_bad(ca);
    wrapper_bad(3);
}
```
#### 为什么它叫完美转发
在`wrapper_bad(T && x)`这种写法里，`T`的推导结果已经编码了调用方传入的是左值还是右值。`std::forward<T>(x)`做的事就是把这个信息还原回表达式的值类别，于是下游函数能像直接接受原始实参那样选择重载，这就是完美转发。

若推广到多个参数是，写法会变成：
> `f(std::forward<Args>(args)...);`

其中`...`会对每个参数分别做一次`forward`，然后再展开成逗号分隔的实参列表。
示例：
```C++
#include <iostream>
#include <utility> // std::forward 必须包含

// 下游重载函数（模拟不同值类别的处理）
void func(int& a) {
    std::cout << "func(int&) —— 处理左值\n";
}

void func(const int& a) {
    std::cout << "func(const int&) —— 处理 const 左值\n";
}

void func(int&& a) {
    std::cout << "func(int&&) —— 处理右值\n";
}

void func(std::string& s) {
    std::cout << "func(string&) —— 处理字符串左值\n";
}

void func(std::string&& s) {
    std::cout << "func(string&&) —— 处理字符串右值\n";
}

template <typename T1, typename T2>
void func(T1&& a, T2&& b) {
    std::cout << "多参数 func：\n";
    func(std::forward<T1>(a)); // 转发第一个参数
    func(std::forward<T2>(b)); // 转发第二个参数
}

// 包装器模板：支持任意数量、任意类型的参数
template <typename... Args>
void wrapper(Args&&... args) {
    // 多参数完美转发写法：逐个转发每个参数
    func(std::forward<Args>(args)...);
}
int main() {
    int x = 10;
    const int cx = 20;
    std::string s = "hello";

    std::cout << "=== 测试单参数 ===" << std::endl;
    wrapper(x);        // 左值 → func(int&)
    wrapper(cx);       // const 左值 → func(const int&)
    wrapper(30);       // 右值 → func(int&&)
    wrapper(s);        // 字符串左值 → func(string&)
    wrapper(std::string("world")); // 字符串右值 → func(string&&)

    std::cout << "\n=== 测试多参数（需扩展 func 为多参数版本） ===" << std::endl;
    // 下面演示多参数转发语法（需配套多参数 func）
    wrapper(x, std::string("tmp"));
}
```
和`std::move`的对比
- `std::move(x)`无条件把`x`转成右值，表达的是“我愿意把资源交出去”。
- `std::forward<T>(x)`有条件地保持调用方传入时的值类别，表达的是“我只做中转，不擅自改变值类别”

注：
- `forward`只能用于这种通过`T&&`接收并由`T`推导得到的形参，不要对普通局部变量随意`forward`。
- 如果把`std::forward<T>(x)`写成`std::move(x)`，就会把左值也强行当右值，很容易制造隐蔽的移动后状态问题。
