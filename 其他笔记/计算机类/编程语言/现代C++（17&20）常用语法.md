# 现代C++17/20 常用语法

## 1. STL预备知识
### 1.1 STL 三件套
```txt
容器:存储数据
算法：处理数据
迭代器：把容器的元素暴露给算法的通用接口
```
#### 关键用法
大多数写法：
```
算法(begin,end,其他参数...)
```
> begin指向第一个元素，end指向最后一个元素**后面**的位置。
### 1.2 区间规则
通常是半开半闭区间`[begin,end)`，也就是包含`begin`指向的元素但是不包含`end`指向的位置。
示例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>x = {3,1,2};
    std::sort(x.begin(),x.end());
    for(int i:x) std::cout << i << std::endl;
    return 0;
}
```
结果：
```
1
2
3
```
- 注：`end()`不是最后元素，不能被解引用。
## 2.C++17常用语法糖
### 2.0 语法糖的解释
**语法糖**是编程语言提供的一种便捷语法—— 它**不增加**语言的核心功能，只是对已有语法的 **“包装”** 或 **“简化”**，让代码写起来更简洁、读起来更直观，但其底层实现和原本的繁琐写法完全一致。
简单来说：**语法糖** = **简化写法** + **等价的底层逻辑**，核心是 “甜化” 代码，降低编写和理解成本，不改变语言的本质能力。
### 2.1 auto
把类型推导交给编译器，使人专注变量语义而不是拼写类型。
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    auto x = 42;
    auto name = std::string("Tom");
    auto nums = std::vector<int>{3,1,2};
    std::cout << x <<" " << name << " " << nums.size();
    return 0;
}
```
输出结果
```
42 Tom 3
```
- 注：右侧表达式易懂时应用`auto`，接口参数和关键变量的类型不明显时，适当显式写类型。
### 2.2 结构化绑定
1. 概念：结构化绑定可以把`pair`,`tuple`或符合要求的结构体**一次性拆解成多个独立变量**
2. 适用范围：
    1. `std::pair`
    2. `std::tuple`
3. `std::pair`简介：
    1. 本质：【二元组】 把两个任意值打包成一个整体
    2. 固定长度(两个元素)，两元素无`键值`关系，无查找功能
4. 例
```C++
#include <iostream>
#include <algorithm>
#include <map>
#include <utility>
int main(){
    std::pair<int,std::string> p{1,"Alice"};
    auto[id,user] = p;
    std::cout << id << " " << user << std::endl;
    std::map<std::string,int> score{{"math",99},{"cs",95}};
    for(const auto &[k,v]:score)
    {
        std::cout << k << ": " << v << std::endl;
    }
    return 0;
}
```
结果
```
1 Alice
cs: 95
math: 99
```
- 注：遍历通常写`const auto &[k,v]`,而非`auto [k,v]`(避免复制元素，过慢)
### 2.3 Lambda
1. 背景：只需要传一个小逻辑的算法
2. `Lambda`是匿名函数对象，常见形式
```
[捕获] (参数) -> 返回类型{函数体}
```
返回类型大多数可以省略。

3. 捕获的直觉
- 值捕获`[x]`：复制一份到`Lambda`里。
- 引用捕获`[&x]`:Lambda里操作的是外部变量本体。
4. 例：
```C++
#include <algorithm>
#include <iostream>
#include <vector>
int main() {
    std::vector<int> a{1,2,3,4,5};
    int bias = 10;
    std::for_each(a.begin(),a.end(),[bias](int &x){x += bias;});
    int even = std::count_if(a.begin(),a.end(),[](int x){return x % 2 == 0;});
    for(int x : a) std::cout << x << " ";
    std::cout << std::endl << "even = " << even << std::endl;
}
```
结果：
```
11 12 13 14 15 
even = 2
```
5. 注：
- `for_each`里形参是`int &x`，会修改容器元素
- `count_if`的Lambda返回布尔值，满足条件就计数
- 二者的一般形式
```
for_each{
    起始迭代器，
    结束迭代器，
    一元操作函数/Lambda
}
```
### 2.4 if与switch初始化语句
1. C++17允许在`if` 和`switch`中先写一个初始化语句，再写条件判断。
目的：希望某个临时变量只在这一段分支逻辑里，不泄露到外层，不和外层变量重名。
2. 用法：
```
if (init; cond) { ... } else { ... }
switch (init; cond) { ... }
```
其中：
- `init`会先执行一次，常见是声明一个变量。
- `cond`里可以使用`init`声明的变量。
3. 例：
```C++
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string,int> mp{{"bob",95},{"alice",90}};
    if(auto it = mp.find("alice");it != mp.end())
    {
        std::cout << it->first << ": " << it->second << "\n";
    }
    else std::cout << "not found";
    // 这里访问不到 it, 因为 it 的作用域被限制在 if 语句里
}
```
```C++
#include <iostream>
int classify(int x)
{
    if(x < 0) return -1;
    if(x == 0) return 0;
    return 1;
}
int main(){
    switch(int c = classify(7);c)
    {
        case -1:
        std::cout << "negative" << std::endl;
        case 0:
        std::cout << "zero" << std::endl;
        case 1:
        std::cout << "positive" << std::endl;
        break;
    }
    return 0;  
}
```
### 2.5 属性与编译器提示
1. 属性写法`[[....]]`不改变语义，但是会让编译器在高风险代码时给出明确提示。
2. 
    1. `[[nodiscard]]`
    希望调用方不要忽略返回值
    例：
    ```C++
    #include <iostream>

    [[nodiscard]] int compute() { return 42; }

    int main() {
        compute(); // 很多编译器会对忽略返回值发出警告
        int x = compute();
        std::cout << x << "\n";
    }
    ```
    会报错

    2. `[[maybe_unused]]`
    确实需要一个变量或参数，但是某些时候用不上
    ```C++
    #include <iostream>

    void log_value([[maybe_unused]] int x) {
    // 调试时可能打印, 线上关闭日志后 x 就会变成未使用
    std::cout << "log enabled\n";
    }
    ```
    3. `[[fallthrough]]`
    明确想要从`switch`的一个`case`继续执行到下一个`case`时
    ```C++
    #include <iostream>

    int main() {
        int level = 1;
        switch (level) {
        case 1:
        std::cout << "basic\n";
        [[fallthrough]];
        case 2:
            std::cout << "advanced\n";
            break;
        default:
            break;
        }
    }
    ```
## 3.STL常用算法
### 3.1 std::for_each
对区间内每个元素执行一次操作

用法 
```C++
std::for_each(first,last,func)
``` 
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>v{1,2,3};
    std::for_each(v.begin(),v.end(),[](int &x){x *= 10;});
    for(int x:v) std::cout << x << " ";
    
}
```
结果
`10 20 30`
### 3.2 std::count_if
统计区间内满足条件的元素个数。它不会修改元素。

用法：
```C++
std::count_if(first,last,pred)
```
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>v{1,2,3,4,5,6};

    int even = std::count_if(v.begin(),v.end(),[](int x) {return x % 2 == 0;});
    std::cout << even;
}
```
结果
`3`

注：谓词`pred`尽量写成纯判断函数, 不要在里面修改外部状态, 否则读代码的人很难确认你是不是在靠副作用实现逻辑。
### 3.3 std::find_if
在区间里找第一个满足条件的元素, 返回**迭代器**。找不到时返回`end`。

用法：`auto it = std::find_if(first,last,pred)`
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
   std::vector<int>v{3,7,11,20};
   auto it = std::find_if(v.begin(),v.end(),[](int x){return x > 10;});
   if(it != v.end()) std::cout << "found: " << *it <<"\n";
   else std::cout << "not found";
}
```
结果
`found: 11`
### 3.4 std::transform
把输入区间的每个元素变换成输出区间的元素。它相当于把一个循环里的赋值写成了算法。

用法：

常见的一元变换写法：
`std::transform(first,last,out,op)`

其中`out`为任意类型的**输出迭代器**

例：计算平方并放入数组
```C++
#include <iostream>
#include <algorithm>
#include <vector>
#include <iterator>
int main(){
   std::vector<int> v{1,2,3,4};
   std::vector<int> out;
   out.reserve(v.size());
   std::transform(v.begin(),v.end(),std::back_inserter(out),[](int x){return x *x;});
   for(int x:out) std::cout << x << " ";
}
```
结果：`1 4 9 16`
- 注`std::back_inserter(out)`会把赋值操作转成 push_back, 因此你不需要提前 `resize` 输出容器。你仍然最好 `reserve` 一下, 避免频繁扩容。
### 3.5 std::accumulate
把一段区间累积成一个值, 常用于求和, 求积, 或把多个值合成一个结果。

用法：
```C++
std::accumulate(first,last,init)
std::accumulate(first,last,init,op)
```
它在内部按顺序做`init = op(init, x)`。
例：
```C++
#include <iostream>
#include <numeric>
#include <vector>
int main(){
   std::vector<int> v{1,2,3,4};
   int s = std::accumulate(v.begin(),v.end(),0);
   std::cout << s << "\n";
}
```
结果：`10`。

`init` 的意义

`init` 不只是初始值, 它还会影响结果类型。例如你希望在整数求和时用更大的类型防止溢出, 可以写 `0LL`:
```C++
long long s = std::accumulate(v.begin(), v.end(), 0LL);
```
### std::sort与二分查找
排序与二分查找经常成套出现: 先排序, 再用二分在对数时间内查找

std::sort
```C++
std::sort(begin, end)
std::sort(begin, end, comp)
```
`comp(a,b)`返回true表示a应该排在b前面。

std::lower_bound

`lower_bound`在已排序区间内找第一个不小于目标值的**位置**。
```C++
auto it = std::lower_bound(begin,end,value)
```
例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int> v{5,1,4,2,3};
    std::sort(v.begin(),v.end());
    int target = 4;
    auto it = std::lower_bound(v.begin(),v.end(),target);
    if(it != v.end() and *it == target) std::cout << "found " << target << "\n";
}
```
结果：`found 4`
- 注：二分查找类算法要求区间已经按同样的规则排好序, 否则结果不可信。
### std::remove_if 与 erase-remove
`remove_if`在很多容器上不会真正缩短长度

对`vector`这类容器，`std::remove_if`会把不需要删除的元素移到前面，返回一个新的逻辑结尾迭代器。用`erase`删除。

用法：
`v.erase(std::remove_if(v.begin(),v.end(),pred),v.end());`

例：删除所有奇数
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int> v{1,2,3,4,5,6};
    v.erase(std::remove_if(v.begin(),v.end(),[](int x){x % 2 == 1};),v.end());
    for(int x : v) std::cout << x << " "; 
    return 0;
}
```
运行结果：

`2 4 6`

解读：
`remove_if`负责把“要保留的元素”挪到前面，`erase`负责删除尾部多余元素

`remove_if`讲解及示例：
```C++
#include <iostream>
#include <algorithm>
#include <vector>
int main(){
    std::vector<int>v {1,2,3,4,5,6};
    auto m = std::remove_if(v.begin(),v.end(),[](int x) { return x % 2 == 1;});
    for(std::vector<int>::iterator it = v.begin();it!=v.end();++it) std::cout << *it << " ";

}
```
运行结果：

`2 4 6 4 5 6`

`remove_if`把需要保留的元素挪到前面后，后几个位置的元素不变(包括值和顺序，容器中元素和原来可不一样)，而不是简单调换位置。
## 4. 值类别与移动语义
**本章是现代C++的核心。**

### 4.1 函数参数传递方式
#### 概念
- 值传递`T`:传进去是一份副本
- 可写引用`T&`:允许函数修改调用方对象
- 只读引用`const T&`:不复制，只读，也能接临时对象
- 右值引用`T&&`:常用于移动语义或转发

注： 
1. 接临时对象：C++规定非常量引用(T&)不能绑定到临时对象。   
    原因：临时对象马上就要销毁，使用非常量引用可能会通过引用修改，导致未定义行为。
2. 右值引用：`T&&`是专门绑定临时对象(右值)的引用

    1. 移动语义——省内存 ：因为临时对象马上就要销毁，可以直接使用其内存，而不需要复制
    例：
    ```C++
    #include <iostream>
    #include <string> 
    // 假设这里有一个大字符串拼接操作
    std::string getLongString() {
        return "This is a very long long long string...";
    }
    int main() {
        // 1. 传统方式：发生拷贝
        // std::string s = getLongString(); 
        // 流程：生成临时对象 -> 拷贝给 s -> 销毁临时对象
        // 2. 使用移动语义（T&&）
        std::string&& s = getLongString(); 
        // 流程：直接把临时对象的“资源指针”移交给 s，不发生拷贝，效率极高
        return 0;
    }
    ```
### 4.2 左值，右值，引用
#### 判别方法：
- 有名字，能反复用的变量表达式，大多数是左值。
- 字面量，临时对象，表达式计算出来的临时结果，大多数是右值。
#### 术语补充
- `lvalue`：左值表达式，例如变量名`x`。
- `prvalue`:纯右值，例如字面量`42`或临时对象`std::string("tmp")`。
- `xvalue`:将亡值，例如`std::move(x)`的结果。它依然引用着某个对象，但通常表示该对象的资源可以被接管。
例：
```C++
#include <iostream>
#include <utility>
void f(int&) {std::cout << "f(int&)\n";}
void f(const int &){std::cout << "f(const int&)\n";}
void f(int &&){std::cout << "f(int&&)\n";}
int main(){
    int x = 10;
    const int cs = 20;

    f(x);
    f(cs);
    f(30);
    f(std::move(x));
}
```
运行结果：
```
f(int&)
f(const int&)
f(int&&)
f(int&&)
```
注：这段代码展示的是重载选择，不是资源真的被搬走
### 4.3 std::move
#### 概念
`std::move`的作用是把表达式转换成右值，更准确地说是`xvalue`。它本身不搬资源，它知识告诉编译器把这个对象当作可被移动的来源，让重载决议更倾向于选择移动构造或移动赋值。
示例：
```C++
#include <iostream>
#include <string>
#include <utility>
int main(){
    std::string a = "hello";
    std::move(a);
    std::cout << a << std::endl;
    std::string b = a;
    std::cout << "b = " << b << " " << "a.size() = "<< a.size() << std::endl;
    b = std::move(a);
    std::cout << "b= " << b << "\n";
    std::cout <<"a.size()= " << a.size() << "\n";
    std::cout << a << std::endl;
    std::cout << &a;
}
```
输出结果：
```
hello
b = hello a.size() = 5
b= hello
a.size()= 0

0xfbd49ff710
```
解释：
`std::move`只是一次类型转换，是后续发生的移动构造或移动赋值以及这些移动操作内部对源对象的处理方式让对象进入`moved-from`状态，拷贝构造和拷贝赋值不会让对象进入`moved-from`状态，单独写一行`std::move(x)`**不会触发任何资源转移**。
#### return时不要写std::move
通常不要写。写`return local`更好
原因：
- 编译器可能做返回值优化，直接在返回位置构造对象。
- 即使不做优化，编译器也可以把局部变量当作可移动对象处理。

例：
```C++
#include <string>
std::string make_name() {
    std::string s = "Tom";
    return s; // 通常写法, 不要强行 std::move(s)
}
```
### 4.4 moved-from对象还能不能用
#### 结论
能用，但只能但一个仍然可析构，可重新赋值的对象
#### 推荐做法
- `move`之后如果还要继续使用，应重新赋值
### 4.5 转发引用与引用折叠
#### 概念
只有当形参写成`T&&`，且`T`需要由实参推导得到时(在模板函数且参数声明为`T&&`且`T`为模板参数)才成立。它的特点时既能绑定左值也能绑定右值，并把值类别的信息编码进`T`的推导结果里。

若修改写法，就不再是转发引用：
- `const T&&`不是转发引用，只能绑定右值
- `std::vector<T>&&`不是转发引用，它只是某个具体类型的右值引用，也只能绑定右值

同样的规则也适用于`auto&&`，例如泛型Lambda的形参或range-for中的`auto&& x`。

引用折叠规则：
| 折叠组合                 | 结果                  | 记忆口诀       |
|--------------------------|-----------------------|----------------|
| 左值引用 + 左值引用      | `T& &` → `T&`         | 有左值则左值   |
| 左值引用 + 右值引用      | `T& &&` → `T&`        | 有左值则左值   |
| 右值引用 + 左值引用      | `T&& &` → `T&`        | 有左值则左值   |
| 右值引用 + 右值引用      | `T&& &&` → `T&&`      | 全右值才右值   |
#### f(T x)按值接收时的推导
当形参是`T x`，`T`的推导更像是做一次按值拷贝需要的类型标准化。
- 实参如果是引用，会先去掉引用再推导`T`
- 实参如果带顶层`const`,推导`T`会去掉这个const
- 数组和函数类型会退化成指针类型

结果是`f(T x)`几乎不可能保留调用方传入的是左值还是右值，也不可能保留顶层`const`。这也是为什么完美转发不可能靠按值参数实现。
####  类型和表达式值类别是两件事
**右值引用不等于右值表达式**
当你在函数体里写`x`时，`x`是一个变量名，它作为表达式属于左值。如果希望把它当作右值去选择右值重载或出发移动语义，需要显示写`std::move(x)`或`std::forward<T>(x)`。
例：
```C++
#include <iostream>
#include <type_traits>

template <class T>
void show_forward(T&&) {
    std::cout << std::boolalpha
              << "forward: T_is_lref=" << std::is_lvalue_reference_v<T>
              << ", T_base_const=" << std::is_const_v<std::remove_reference_t<T>> << "\n";
}

template <class T>
void show_value(T) {
    std::cout << std::boolalpha
              << "value:   T_is_lref=" << std::is_lvalue_reference_v<T>
              << ", T_base_const=" << std::is_const_v<std::remove_reference_t<T>> << "\n";
}

int main() {
    int x = 1;
    const int cx = 2;

    show_forward(x);
    show_value(x);

    show_forward(cx);
    show_value(cx);

    show_forward(3);
    show_value(3);
}
```
输出：
```
forward: T_is_lref=true, T_base_const=false
value:   T_is_lref=false, T_base_const=false
forward: T_is_lref=true, T_base_const=true
value:   T_is_lref=false, T_base_const=false
forward: T_is_lref=false, T_base_const=false
value:   T_is_lref=false, T_base_const=false
```
注：`T&&`是一种省略参数的写法，是合法的
### std::forward
<span style="color: #FF0000; font-size: 18px;">此处很重要</span>

一句话总结`std::forward`的目标：让下游函数看到的值类别尽量等于调用方传进来的值类别。

注：值类别是表达式的属性，不是类型的属性。即使一个变量的类型是右值引用，它只要有名字，这个名字再表达式规则里就是左值。

例1：
```C++
#include <iostream>
#include <type_traits>

void sink(int &) {std::cout << "sink(int&)\n";}
void sink(int &&) {std::cout << "sink(int &&)\n";}

int main() {
    int && rr = 3;  //rr 的类型是 int &&
    sink(rr);       //rr 这个名字是左值表达式
    sink(std::move(rr));    //move把表达式转换成右值
}
```
解释：`rr` 的类型是`int &&`,但`rr`作为表达式仍然走左值重载，这不是模板特例，而是 C++ 表达式规则。

模板里的形参`x`也是同样道理：用`T&&`借助==接住一个右值后，`x`的声明类型可能是`U&&`，但在函数体里直接写`x`时，它依然是左值表达式，所以需要`forward`才能恢复成合适的值类别。
#### 第一步 先观察坏掉的地方
下面这段代码里，调用方传了右值`3`,但`sink(x)`仍然走左值重载。原因是`x`是一个有名字的变量表达式。
```C++
#include <iostream>
void sink(int&) { std::cout << "sink(int&)\n"; }
void sink(int&&) { std::cout << "sink(int&&)\n"; }
template <typename T>
void wrapper_bad(T&& x) {
    sink(x); // x 有名字, 表达式永远是左值
}

int main() {
    int a = 1;
    wrapper_bad(a);
    wrapper_bad(3);
}
```
#### 第二步 T 是怎么推导出来的
现在看`wrapper_bad(T && x)`这一行。这里的`T&&`处于模板推导语境，所以是转发引用，推导规则为：
- 如果实参是左值`U`，那么`T`推导为`U&`
- 如果实参是右值`U`，那么`T`推导为`U`

加上引用折叠，结果如下：
|调用方式            |T 的推导结果        |形参类型 T&& 的折叠结果|
|-------------------|--------------------|-------------|
|wrapper_bad(a)     |int&               |int&|
|wrapper_bad(ca)     |const int&         |const int&
|wrapper_bad(3)      |int                |int&&

注：这张表只是说类型，并未说表达式。即使`x`的类型是`int&&`，在函数体里写`x`时，依然是左值表达式。
#### 第三步 forward的语义
`std::forward<T>(x)`的本质是一次条件转换：
- 当`T`是左值引用类型时，它把`x`当作左值返回
- 当`T`不是左值引用类型时，它把`x`当作右值返回

可以理解为：
>`static_cast<T&&>(x)`

这行`cast`能起作用的关键还是引用折叠
|               |                 |                                                           |
|---------------|-----------------|---------|
|如果 T = U&    |那么 T&& 折叠成 U&|     static_cast<T&&>(x) 等价于 static_cast<U&>(x)|
|如果 T = U     |那么 T&& 就是 U&& |    static_cast<T&&>(x) 等价于 static_cast<U&&>(x)|

因此`forward<T>(x)`利用`T`的推导结果把`x`转回合适的值类别

#### 第四步 正确写法与运行结果
```C++
#include <iostream>
#include <type_traits>

void sink(int &) {std::cout << "sink(int&)\n";}
void sink(const int&) {std::cout << "sink(cons int&)\n";}
void sink(int &&) {std::cout << "sink(int &&)\n";}

template<typename T>
void wrapper_bad(T && x){sink(std::forward<T>(x));}
int main() {
    int a = 1;
    const int ca = 2;  

    wrapper_bad(a);
    wrapper_bad(ca);
    wrapper_bad(3);
}
```
#### 为什么它叫完美转发
在`wrapper_bad(T && x)`这种写法里，`T`的推导结果已经编码了调用方传入的是左值还是右值。`std::forward<T>(x)`做的事就是把这个信息还原回表达式的值类别，于是下游函数能像直接接受原始实参那样选择重载，这就是完美转发。

若推广到多个参数是，写法会变成：
> `f(std::forward<Args>(args)...);`

其中`...`会对每个参数分别做一次`forward`，然后再展开成逗号分隔的实参列表。
示例：
```C++
#include <iostream>
#include <utility> // std::forward 必须包含

// 下游重载函数（模拟不同值类别的处理）
void func(int& a) {
    std::cout << "func(int&) —— 处理左值\n";
}

void func(const int& a) {
    std::cout << "func(const int&) —— 处理 const 左值\n";
}

void func(int&& a) {
    std::cout << "func(int&&) —— 处理右值\n";
}

void func(std::string& s) {
    std::cout << "func(string&) —— 处理字符串左值\n";
}

void func(std::string&& s) {
    std::cout << "func(string&&) —— 处理字符串右值\n";
}

template <typename T1, typename T2>
void func(T1&& a, T2&& b) {
    std::cout << "多参数 func：\n";
    func(std::forward<T1>(a)); // 转发第一个参数
    func(std::forward<T2>(b)); // 转发第二个参数
}

// 包装器模板：支持任意数量、任意类型的参数
template <typename... Args>
void wrapper(Args&&... args) {
    // 多参数完美转发写法：逐个转发每个参数
    func(std::forward<Args>(args)...);
}
int main() {
    int x = 10;
    const int cx = 20;
    std::string s = "hello";

    std::cout << "=== 测试单参数 ===" << std::endl;
    wrapper(x);        // 左值 → func(int&)
    wrapper(cx);       // const 左值 → func(const int&)
    wrapper(30);       // 右值 → func(int&&)
    wrapper(s);        // 字符串左值 → func(string&)
    wrapper(std::string("world")); // 字符串右值 → func(string&&)

    std::cout << "\n=== 测试多参数（需扩展 func 为多参数版本） ===" << std::endl;
    // 下面演示多参数转发语法（需配套多参数 func）
    wrapper(x, std::string("tmp"));
}
```
结果：
```
=== 测试单参数 ===
func(int&) —— 处理左值
func(const int&) —— 处理 const 左值
func(int&&) —— 处理右值
func(string&) —— 处理字符串左值
func(string&&) —— 处理字符串右值

=== 测试多参数（需扩展 func 为多参数版本） ===
多参数 func：
func(int&) —— 处理左值
func(string&&) —— 处理字符串右值
```
和`std::move`的对比
- `std::move(x)`无条件把`x`转成右值，表达的是“我愿意把资源交出去”。
- `std::forward<T>(x)`有条件地保持调用方传入时的值类别，表达的是“我只做中转，不擅自改变值类别”

注：
- `forward`只能用于这种通过`T&&`接收并由`T`推导得到的形参，不要对普通局部变量随意`forward`。
- 如果把`std::forward<T>(x)`写成`std::move(x)`，就会把左值也强行当右值，很容易制造隐蔽的移动后状态问题。
## 5. 模板里的常用技巧
模板这一章的目标很明确：让你在读代码时能知道作者像表达什么，在写工具函数时能用上最实用的两三找。这里不追求把模板讲成一门独立学科，而是先抓住`if constexpr`与参数包这类高频写法。
### 5.1 if constexpr
背景

普通`if`的两个分支都会先通过语法检查，模板里很容易因此报一堆无关错误。

概念

`if constexpr`在编译期选择分支，未命中的分支不会实例化。

例：
```C++
#include <iostream>
#include <type_traits>

template <typename T>
void print_type_info(const T& x) {
    if constexpr (std::is_integral_v<T>) {
        std::cout << "integral: " << x << "\n";
    } else if constexpr (std::is_floating_point_v<T>) {
        std::cout << "floating: " << x << "\n";
    } else {
        std::cout << "other\n";
    }
}

int main() {
    print_type_info(10);
    print_type_info(3.14);
    print_type_info("hi");
}
```
结果：
```
integral: 10
floating: 3.14
other
```
解释：
当实例化 print_type_info(10) 时，T = int，此时第二和第三个分支不会被实例化，编译器不会检查是否有错误。
### 5.2可变参数模板与折叠表达式
本节解决两类高频需求。
1. 把任意多个参数原样转交给下游函数 -- 参数包的声明与展开
2. 把一串参数合并成一个表达式 -- 求和或依次输出，这对应折叠表达式。
#### 5.2.1 参数包展开
把参数包想成一个编译期列表：`Args...`是一组类型，`args...`是一组值。看到`...`时先判断它在做什么：如果跟在名字后面，通常时在声明函数包，例如`template <class... Args>`和`void f(Args&&... args)`;如果跟在包含参数包的模式后面，通常是在展开参数包，例如`std::tuple<Args...>`或`g(args...)`。

展开的核心规则是写出一个模式，再在后面加`...`。编译器会把这份模式复制多次，每次用参数包里的下一个元素替换，然后用逗号把结果拼成列表。
例：
```C++
#include <string>
#include <tuple>
#include <vector>

template <class... Args>
using VecTuple = std::tuple<std::vector<Args>...>;

VecTuple<int, std::string> t; // 等价于 tuple<vector<int>, vector<string>> 
```
这里模式是`std::vector<Args>`。当`Args...`有两个元素时，模式就会复制两份，生成`std::vector<int>,std::vector<std::string>`。如果一个模式里同时出现多个参数包，它们必须能一一对应，也就是长度相同，否则无法展开。

值参数的展开完全一样，只是把类型替换成值。下面这个例子里，`sink(args...)`展开后就是把每个参数按原顺序传给`sink`:
```C++
#include <iostream>

void sink(int a,int b,int c)
{
    std::cout << a << " " << b << " " << c << "\n";
}

template<typename... Args>
void forward_to_sink(Args... args)
{
    sink(args...);
}

int main(){

    forward_to_sink(1,2,3);
}
```
在工程里还会频繁看到`sizeof...(Args)`或`sizeof...(args)`。它返回参数包元素个数，常用于写边界处理和静态断言，例如限制参数至少有一个或者在调试时输出数量。

完美转发时**语法长**的作用

例：
```C++
#include <utility>

template <typename F, typename... Args>
decltype(auto) call(F&& f, Args&&... args) {
    return std::forward<F>(f)(std::forward<Args>(args)...);
}
```
重点不是语法长，而是每个参数都要各自保持值类别：`Args&&...`让每个参数分别推导，`atd::forward<Args>(args)...`让每个参数分别转回合适的值类别，然后再展开成实参列表。**牺牲简洁性的目的：换 “调用准确性” 和 “零拷贝效率”，这是编写通用、高效包装函数的必要代价。**

#### 5.2.2 折叠表达式
当你不是把参数继续传给下游函数，而是要把他们合并成一个结果，就要用到折叠表达式。他的作用时把一个参数包在语法层面折成一个表达式。为了看清四种形态，可以假设`args`展开后依次是`a,b,c`,`op`为操作符:
|                       |                        |
|-----------------------|------------------------|
|(... op args)          | 等价于((a op b) op c)  |
|(args op ...)|等价于(a op (b op c))|
|(init op ... op args)| 等价于(((init op a) op b) op c)|
|(args op ... op init)|等价于(a op (b op (c op init)))
左折叠和右折叠并不总是等价。用减法可以直观看出差别。
```C++
#include <iostream>

template <class...Args>
auto left_minus(Args...args)
{
    return (... - args);
}
template <class...Args>
auto right_minus(Args...args)
{
    return (args - ...);
}
int main(){
    std::cout << left_minus(10,1,2) << "\n";
    std::cout << right_minus(10,1,2) << "\n";
}
```
结果：
```
7
11
```
一元折叠`(... op args)`和`(args op ...)`要求参数包非空，否则无法生成表达式。带初值的二元折叠可以支持空参数包，空时结果就是初值。

这些规则落在具体代码上，会得到非常常见的两类写法：
1. 用`+`等运算符做聚合。
2. 用逗号运算符“依次执行”。
```C++
#include <iostream>
#include <utility>

template <typename... Args>
auto sum(Args... args) {
    return (0 + ... + args);
}

template <typename... Args>
void log_line(Args&&... args) {
    ((std::cout << std::forward<Args>(args) << " "), ...);
    std::cout << "\n";
}

template <class... Bools>
bool all_true(Bools... bs) {
    return (true && ... && bs);
}

int main() {
    std::cout << sum(1, 2, 3, 4) << "\n";
    std::cout << sum() << "\n";
    log_line("id=", 7, "score=", 98.5);
    std::cout << std::boolalpha << all_true(true, true, false) << "\n";
    std::cout << std::boolalpha << all_true() << "\n";
}
```
`log_line`里的这一句是对逗号运算符做折叠。可理解成按从左到右顺序依次执行。

注：
- 不要把`...`当成魔法符号，最有效的排错方式是把他们展开成等价形式写出来
- 转发场景要写`std::forward<Args>(args)...`，不要写成`std::forward<Args...>(args...)`
- 如果使用`std::move(args)...`代替forward,会把左值也强行当右值，容易引入意外移动。
- 折叠表达式里如果依赖求值顺序，有效使用`,`折叠或短路运算符折叠，不要寄托`+`这类运算符上。
### 5.3 decltype 与 decltype(auto)
本节的关键词是两件基础的事：什么是名字，什么是表达式。`decltype`和`decltype(auto)`都依赖表达式规则，其中`decltype`还对“裸名字”做了一个特殊处理。
#### 5.3.1 decltype的规则
`decltype(e)`用来得到表达式`e`的类型，但不是简单的“把类型抄下来”，它的规则可以按两步记：

第一步先看`e`是不是一个裸名字。这里的裸名字指未加括号的变量名，或未加括号的成员访问表达式。
- 如果是裸名字，`decltype(e)`得到的是这个名字的声明类型
- 如果不是裸名字，就进入第二步
第二步看表达式的值类别：
- `e`是左值表达式`decltype(e)`是`T&`
- `e`是将亡值表达式，`decltype(e)`是`T&&`
- `e`是纯右值表达式，`decltype(e)`是`T`

这就是为什么`x`和`(x)`不一样，`x`是裸名字，会触发第一步的特殊处理，而`(x)`不是裸名字，会按第二步的值类别规则走，因为`(x)`是左值表达式，所以得到引用类型。
```C++
#include <type_traits>
#include <utility>

int main() {
    int x = 0;
    const int cx = 1;

    static_assert(std::is_same_v<decltype(x), int>); // 裸名字，取int
    static_assert(std::is_same_v<decltype((x)), int&>); // 左值，推导为int&

    static_assert(std::is_same_v<decltype(cx), const int>); //裸名字，取const int
    static_assert(std::is_same_v<decltype((cx)), const int&>); //const 左值表达式，推导为const int &

static_assert(std::is_same_v<decltype(std::move(x)), int&&>); //将亡值，推导为int&&
}
```
#### 5.3.2
`decltype(auto)`是一种占位写法，它的意思是:用`decltype(expr)`的规则去推导类型,精确保留表达式的`引用/const`属性，和`decltype(expr) v = expr;`语义几乎一致，这在写模板包装器时很有价值。

`auto`则使用模板类型推导的规则，会去掉引用和顶层const，更像按值接收。 
```C++
#include <iostream>
#include <type_traits>

int main() {
    int x = 1;
    const int cx = 2;

    auto a = x;
    auto b = (x);
    decltype(auto) c = x;
    decltype(auto) d = (x);

    static_assert(std::is_same_v<decltype(a), int>); // int
    static_assert(std::is_same_v<decltype(b), int>); // int
    static_assert(std::is_same_v<decltype(c), int>); // int 裸名字
    static_assert(std::is_same_v<decltype(d), int&>); // int & 推导为 T& d为x别名

    decltype(auto) e = cx;
    decltype(auto) f = (cx);
    static_assert(std::is_same_v<decltype(e), const int>);
    static_assert(std::is_same_v<decltype(f), const int&>);

    d = 10;
    std::cout << x << "\n";
}
```
结果：
>10

既然`decltype(auto)`这么精确，它在工程里的作用？

最常见的用法时写一层很薄的包装器，这层包装器不想改变下游API的语义，只是想加一点额外逻辑，例如打日志，做边界检查，做统一统计。只要希望调用点仍能拿到引用，`decltype(auto)`就比`auto`合适。

```C++
#include <cstddef>
#include <iostream>
#include <vector>

template <class Container>
auto elem_copy(Container& c, std::size_t i) {
    return c[i];
}

template <class Container>
decltype(auto) elem_exact(Container& c, std::size_t i) {
    return c[i];
}

int main() {
    std::vector<int> v{1, 2, 3};

    auto a = elem_copy(v, 0);   // a 是 int, 拷贝了一份
    auto& b = elem_exact(v, 0); // b 是 int&, 引用容器里的元素

    b = 100;
    std::cout << v[0] << "\n";

    // elem_copy(v, 0) = 10;  // 这行无法编译, 因为 elem_copy 返回的是临时值
     elem_exact(v, 1) = 10;
}
```
结果：
```
100
10
```
再看一个更贴近模板库代码的场景，你想写一个通用的`call`包装器，让它把任意可调用对象的返回值原样传回去。只要写有返回的时引用，`auto`就会把引用丢掉，而`decltype(auto)`能把这个引用保留下来，调用点也就能继续赋值。
```C++
#include <iostream>
#include <string>
#include <utility>

std::string& identity(std::string& s) {
    return s;
}

template <class F, class... Args>
auto call_copy(F&& f, Args&&... args) {
    return std::forward<F>(f)(std::forward<Args>(args)...);
}

template <class F, class... Args>
decltype(auto) call_exact(F&& f, Args&&... args) {
    return std::forward<F>(f)(std::forward<Args>(args)...);
}

int main() {
    std::string s = "Alice";

    auto a = call_copy(identity, s);
    a = "X";
    std::cout << s << "\n";

    call_exact(identity, s) = "Bob";
    std::cout << s << "\n";
}
```

返回类型里为什么常见decltype(auto)

当你写一个中转调用的模板函数时，你往往把下游函数的返回类型原样传回去，包括引用与const。`decltype(auto)`很适合这个场景，因为它会按返回表达式的值类别推导返回类型。

因为它很精确，所以需要对返回表达式更谨慎。下面这个反例能编译，但返回的时局部变量引用，调用方拿到的是悬空引用。

```C++
decltype(auto) bad() {
    int x = 0;
    return (x);
}
```

如果写成`return x`，返回类型就会变成按值返回，行为也更符合直觉。工程里使用`decltype(auto)`的一个常见原则是：它适合写转发包装器与库工具，但在普通业务函数里，只要按值返回就能表达清楚，不要为了保留引用而增加悬空风险。
### 5.4 type_traits 与 static_assert
模板代码经常需要根据类型做分支，或者在编译期拒绝某些用法，`<type_traits>`提供了一大批类型性质与类型变换工具，`static_assert.`则负责在编译期把不符合要求的用法拦下来。

**static_assert**
```C++
#include <type_traits>

template <class T>
void only_integral(T){
    static_assert(std::is_integral_v<T>,"T must be integral";)
}
```
当用`only_integral(3.14)`调用时，编译期就会报错，而不是等到运行时。

**常用性质判断**
在工程代码里最常见的时这些`_v`结尾的变量模板：
- `std::is_integral_v<T>`
- `std::is_floating_point_v<T>`
- `std::is_same_v<A,B>`
- `std::is_reference_v<T>`

**常用类型变换**
类型变换工具把一个类型转换成另一个更便于处理的类型
- `std::remove_reference_t<T>` 去掉引用
- `std::remove_const_t<T>` 去掉顶层const
- `std::decay_t<T>` 做一组常见衰变，常用于把类型变成“像按值传递那样”的形态

当看到模板里写了很多`_t`，通常就是在做这种类型标准化。

注：`std::decay_t<T>` 是 C++11 引入的类型萃取工具（位于 <type_traits> 头文件），核心作用是模拟 C++ 中按值传递参数时的类型 “衰变” 规则—— 把复杂类型（引用、数组、函数、`const/volatile` 限定等）转换成 “传值时的原始形态”，最终得到一个可拷贝、无引用、无 cv 限定（除非是底层 `const`）、数组 / 函数转为指针的 “纯值类型”。
### 5.5 enable_if 与替代失败
在C++20之前，约束模板参数最常见的手段就是`std::enable_if`。它经常以一种看起来很诡异的形式出现，例如模板参数列表里多出来一段`std::enable_if_t<...,int> = 0`。
#### 5.5.1 从一个例子开始
下面的`primer_number`有两份候选实现，一份只接整数类型，一份只接浮点数类型。调用点看起来很普通，但重载决议回根据`T`的性质自动选择能用的那一份。
```C++
#include <iostream>
#include <type_traits>

template <class T, std::enable_if_t<std::is_integral_v<T>, int> = 0>
void print_number(T x) {
    std::cout << "integral: " << x << "\n";
}

template <class T, std::enable_if_t<std::is_floating_point_v<T>, int> = 0>
void print_number(T x) {
    std::cout << "floating: " << x << "\n";
}

int main() {
    print_number(7);
    print_number(3.14);
}
```
结果：
```
integral: 7
floating: 3.14
```
下面拆开解释`std::enable_if_t<...,int> = 0`
#### 5.5.2 enable_if 和 =0 的含义
`std::enable_if_t<cond,x>`本身可以理解成一种条件类型：
- `cond`为真时，结果是类型`x`
- `cond`为假时，结果没有类型
 
 而 `=0`不是赋值，而是第二个模板参数的默认值。它的作用时让调用点不需要手动写出这个额外参数，否则就得写`print_number<int ,0>(7)`这种完全没有语义价值的模板实参。

把它按条件展开更直观：
- 当`T=int`时，`std::enable_if_t<true,int>`等价于`int`,于是这一行等价于`template<class T,int = 0> void print_number(T)`
- 当`T = double`时，`std::enable_if_t<false,int>`没有类型
这份候选在替换模板参数时失败，编译器回把它从候选集合里移除

默认值写成`0`只是常见习惯，写成`= 1`也一样，这个参数只是一个用来挂条件的占位符。

如果不喜欢` = 0`的这种非类型模板参数写法，也可以用一个默认类型参数来承载enable_if，逻辑一致。
```C++
template <class T, class = std::enable_if_t<std::is_integral_v<T>>>
void print_number(T);
```
#### 5.5.3 替代失败最核心的用处
核心价值是编译期分流：可以写多份候选实现，让编译器根据类型性质或某个表达式是否成立，自动选择能用的那一份，而不是在函数体里写一堆if-else或让代码直接报硬错误。
它在工程里最常解决三类问题：
- 接口约束：只对满足条件的类型开放函数
- 可用性探测：有某个成员或操作符就走更好的实现，没有就退回通用实现
- 降低报错噪声，让错误停在调用点，不要深入模板实现细节后爆出大量诊断

替代失败有一个边界条件：它只在模板参数替换的立即上下文里生效，例如模板参数列表，返回类型的`decltype`，默认参数类型等。如果错误发生在函数体内部，通常是硬错误，不会自动退回到其他重载。

## 6.标准库工具类型

### 6.1 std::optional
#### 背景
很多函数存在失败情况。用特殊值表示失败容易含混。
#### 概念
`std::optional<T>`表示要么有一个`T`,要么没有值
#### 常用接口
- `opt.has_value()`或`if(opt)`判断是否有值
- `*opt`或`opt->member`访问内部值，前提是有值
- `opt.value()`访问内部值，没值会抛异常
- `opt.value_or(default_value)`没值时返回默认值
- `opt.reset()`清空optional
- `opt.emplace(args...)`直接在optional内部构造一个值
例：
```C++
#include <cstddef>
#include <iostream>
#include <optional>
#include <stdexcept>
#include <string>
#include <string_view>

std::optional<int> parse_int(std::string_view s) //std::string_view只是只读视图(6.3会讲到)，不拷贝数据，效率更高，适合做函数参数 
{
    if (s.empty()) return std::nullopt;

    try {
        std::string tmp(s);
        std::size_t idx = 0;
        int value = std::stoi(tmp, &idx); //std::stoi读到**非数字就停止**
        if (idx != tmp.size()) return std::nullopt;
        return value;
    } catch (const std::invalid_argument&) {
        return std::nullopt;
    } catch (const std::out_of_range&) {
        return std::nullopt;
    }
}
int main() {
    if (auto v = parse_int("18")) {
        std::cout << *v << "\n";
    }

    if (auto v = parse_int("18abc"); !v) {
        std::cout << "invalid\n";
    }
}
```
结果：
```
18
invalid
```
注：
- `parse_int`返回`std::nullopt`表示失败，返回`int`表示成功。
- `if(auto v = parse_int("18")){ ... }`利用的是`optional`的布尔语义：有值为真，没值为假
- 当你写`*v`时，你是在取出内部的`int`值，所以必须先判断`v`是否有值。

例：
```C++
#include <iostream>
#include <optional>

int main() {
    std::optional<int> a = 7;
    std::optional<int> b = std::nullopt;

    std::cout << a.value_or(0) << "\n";
    std::cout << b.value_or(0) << "\n";
}
```
注：
- 忘记判空就`*opt`或`opt.value()`。
- 把`optional`当作“可能为空的引用”。`optional<T>`持有的使一个`T`的值，不会指向外部对象。
### 6.3 std::variant
#### 概念
`std::variant<A,B>`在任意时刻保存A或B的其中一个值
#### 常用接口
- `v.index()`返回当前保存的是第几个选项
- `std::holds_alternative<T>(v)`判断当前是否是T
- `std::get<T>(v)`取出T，取错会抛`std::bad_variant_access`
- `std::get_if<T>(&v)`安全版本，成功则返回指针，失败返回空指针
- `std::visit(visitor,v)`统一访问，不用手写一堆if-else
例：
```C++
#include <iostream>
#include <string>
#include <variant>

int main() {
    std::variant<int, std::string> v = 7;
    std::visit([v](const auto& x) { std::cout << x << "\n";std::cout<< v.index() << std::endl; }, v);
    // std::visit自动把variant内部当前存储的值取出来，赋值给x
    v = std::string("hello");
    std::visit([](const auto& x) { std::cout << x << "\n"; }, v);
}
```
结果：
```
7
0
hello
```
例：用 get_if做安全访问
```C++
#include <iostream>
#include <string>
#include <variant>

int main() {
    std::variant<int, std::string> v = std::string("hi");

    std::cout << "string size = " << std::get<std::string>(v).size() << "\n";

    // 错误：std::cout << "string size = " << std::get<int>(v) << "\n";
    /*
    编译可过，运行时会出现 string size = terminate called after throwing an instance of 'std::bad_variant_access'
  what():  std::get: wrong index for variant
    */

    if (auto p = std::get_if<std::string>(&v)) {
        std::cout << "string size = " << p->size() << "\n";
    } else {
        std::cout << "not a string\n";
    }
}
```
结果：
```
string size = 2
string size = 2
```
### 6.3 std::string_view
#### 概念
`std::string_view`只是一段字符串的视图，它不拥有内存。
#### 视图的本质
可以理解为两样东西的组合
- 一个指向字符数据的指针
- 一个长度

因此拷贝非常便宜，但是也不会替你管理底层内存

#### 适用场景
- 函数只读参数
- 避免不必要拷贝

#### 常用接口
- `sv.size()`获取长度
- `sv.data()`获取底层指针，但不保证以`\\0`结尾
- `sv.substr(pos,n)`取子串视图
- `sv.remove_prefix(n)`和`sv.remove_suffix(n)`原地移动视图边界
例：
```C++
#include <iostream>
#include <string>
#include <string_view>

void print_prefix(std::string_view sv) {
    std::cout << sv.substr(0, 3) << "\n";
}

int main() {
    std::string s = "abcdef";
    print_prefix(s);
    print_prefix("xyz123");
}
```
结果：
```
abc
xyz
```
##### 反例1 绑定到临时string
下面这行代码看起来很正常, 但`std::string("abc")` 是临时对象, 这一行结束后它就销毁了,`sv`会悬空:
```C++
#include <string>
#include <string_view>
#include <iostream>
int main() {
    std::string_view sv = std::string("abc");
    (void)sv;
    std::cout << sv;
}
```
结果:
```
abc
```
解释：

虽然可以输出`abc`，但是这是未定义行为，内存虽然被释放，但是还没被覆盖，所以**看起来**能用。
例：
```C++
#include <iostream>
#include <string>
#include <string_view>

void dirty_stack() {
    int arr[100] = {0}; // 栈内存写得更多
}

int main() {
    std::string_view sv;
    {
        std::string tmp = "abcdefghijklmnopqrstuvwxyz"; // 更长
        sv = tmp;
    }
    dirty_stack();
    std::cout << sv << std::endl; // 必乱码 / 崩溃
}
```
结果：
```
纆嬶?P嬶?qrstuvwxyz
```
前半段乱码是因为堆内存被释放后写入了调试 / 管理数据，后半段保留是因为还没来得及被覆盖。这正是未定义行为 “不可预测” 的典型表现。
##### 反例2 返回悬空视图
```C++
#include <iostream>
#include <string>
#include <string_view>

std::string_view bad_view() {
    std::string s = "abc";  // s 是局部变量
    return s;               // 返回指向 s 的视图
} // 🔥 s 在这里被销毁，视图悬空！
std::string bad_view2()
{
     std::string s = "abc";  // s 是局部变量
    return s; 
}

int main() {
    std::string_view sv = bad_view();  // 拿到悬空视图
    std::string s = bad_view2();
    std::cout << "sv = " << sv << std::endl;
    std::cout << "s = " << s << std::endl;
}
```
结果：
```
sv = 
s = abc
```
通常有两种修正思路：
- 让被引用的字符串活得更久，例如由调用方持有`std::string`
- 直接返回`std::string`，让返回值拥有内存。

## 7.智能指针与所有权
本章的核心不是“语法怎么写”，而是“所有权怎么表达”。如果你能清楚回答每个对象是谁负责释放，那么很多内存问题就会自然消失。
### 7.1 std::unique_ptr

概念

`unique_ptr`表示独占所有权，不能拷贝，只能移动。

常用接口
- `p.get()`取裸指针，不转移所有权
- `p.release()`交出所有权并返回裸指针，之后`p`变空，你需要自己负责delete
- `p.reset(new_ptr)`释放旧对象并接管新对象，也可以`reset()`直接清空
- `if(p)`判断是否持有对象

一句话规则

谁持有`unique_ptr`，谁负责释放对象。所有权要么在一个点上明确转移，要么就一直由一个变量管理。
例：
```C++
#include <iostream>
#include <memory>
#include <utility>

struct Worker {
    void run() const { std::cout << "working\n"; }
};

std::unique_ptr<Worker> make_worker() {
    return std::make_unique<Worker>();
}

int main() {
    auto p = make_worker();
    p->run();

    auto q = std::move(p);
    if (!p) std::cout << "p moved-from\n";
    q->run();
}
```
结果：
```
working
p moved-from
working
```
解释：
- `make_worker`返回`unique_ptr`时，所有权从函数内部转移到调用方。
- `auto q = std::move(p);`之后，所有权就是从`p`转移到`q`,所以`p`变空
#### std::make_uniuqe函数
C++14引入的创建`std::unique_ptr`智能指针的标准函数，专门用来安全，简洁地创建独占智能指针，替代手动写`new`。

`std::make_unique<class>(args...)`等价于做了三件事：
1. 自动`new`一个对象
2. 自动把对象包装进`std::unique_ptr`
3. 自动管理生命周期，离开作用域就释放，不会内存泄漏

好处：**安全**，**清晰**，**符合现代C++实践**

`std::make_unique`还可以直接创建数组
- 创建动态数组：`make_unique<Worker[]>(5)`

例：把所有权传给函数

把`unique_ptr`作为参数按值传递，表达的是被调函数要接管所有权，调用方必须显式`std::move`。
```C++
#include <iostream>
#include <memory>
#include <utility>

struct Worker {
    void run() const { std::cout << "working\n"; }
};

void consume(std::unique_ptr<Worker> p) {
    if (p) p->run();
}

int main() {
    auto p = std::make_unique<Worker>();
    consume(std::move(p));
    // unique_ptr不可拷贝，只能移动，必须用std::move转移所有权
    // 对象所有权从main的p转移到consume的形参p，函数结束后形参p销毁，对象内存自动释放
    if (!p) std::cout << "p is empty\n";
}
```
结果
```
working
p is empty
```
### 7.2 std::shared_ptr
概念

`shared_ptr`表示共享所有权，用引用计数管理生命周期。

常用接口：
- `p.use_count()`查看当前的引用计数，主要用于调试理解(有几个在共享这个内存)
- `p.reset()`释放一次共享引用，计数减一
- `if(p)`判断是否为空
- `p.get()`取裸指针，不改变计数

传参建议
- 只读使用：优先传`const std::shared_ptr<T>&`或直接传`T*`或`T&`取决于你是否要延长生命周期
- 需要共享所有权：传`std::shared_ptr<T>`按值，表示你要把这份所有权保存下来
例：
```C++
#include <iostream>
#include <memory>

struct Task {
    ~Task() { std::cout << "Task destroyed\n"; }
};

int main() {
    auto a = std::make_shared<Task>();
    std::cout << "count=" << a.use_count() << "\n";

    {
        auto b = a;
        std::cout << "count=" << a.use_count() << "\n";
    }

    std::cout << "count=" << a.use_count() << "\n";
}
```
结果：
```
count=1
count=2
count=1
Task destroyed
```
解读

`auto b = a`会让引用计数加一。离开作用域后`b`被销毁，计数减一。最后`a`销毁时计数归零，对象才真正析构。

注：
- `shared_ptr`能保证多线程下安全地共享/释放指针，引用计数不会算错，对象不会被提前释放或漏释放
- `shared_ptr`不能保证多个线程同时读写被管理对象时数据不会错乱，可能会有并发问题。
- **总结**：`shared_ptr`本身的引用计数是线程安全的，但是`shared_ptr`管理的对象不是线程安全的

示例：
```C++
#include <iostream>
#include <memory>
#include <thread>
#include <vector>

using namespace std;

// 被 shared_ptr 管理的对象
struct MyData {
    int count = 0;  // 普通变量，不是线程安全
};

// 线程安全：shared_ptr 拷贝/销毁
void test_shared_ptr_safe() {
    auto sp = make_shared<MyData>();
    vector<thread> threads;

    // 10 个线程同时拷贝 shared_ptr
    for (int i = 0; i < 10; ++i) {
        threads.emplace_back([sp]() {
            // 拷贝 shared_ptr：引用计数 +1（原子操作，线程安全）
            auto copy = sp;
        });
    }

    for (auto& t : threads) t.join();

    // 最终引用计数一定正确 = 1
    cout << "shared_ptr 引用计数安全：use_count = " << sp.use_count() << endl;
}

// 线程不安全：对象内部数据被并发修改
void test_object_unsafe() {
    auto sp = make_shared<MyData>();
    vector<thread> threads;

    // 10 个线程同时修改对象里的 count
    for (int i = 0; i < 10; ++i) {
        threads.emplace_back([sp]()
        {
            for (int j = 0; j < 100000; ++j) {
                sp->count++;  // ❌ 非原子操作！数据竞争！
            }
        });
    }

    for (auto& t : threads) t.join();

    // 理论应该 = 10*100000 = 1000000
    // 实际一定小于这个数
    cout << "对象数据不安全：最终 count = " << sp->count << endl;
}

int main() {
    test_shared_ptr_safe();
    test_object_unsafe();
    return 0;
}
```
第一次结果：
```
shared_ptr 引用计数安全：use_count = 1
对象数据不安全：最终 count = 160515
```
第二次结果
```
shared_ptr 引用计数安全：use_count = 1
对象数据不安全：最终 count = 186599
```
我们可以观察到，引用计数的结果是一样的，而count的结果每次都不一样，因为多线程是同时跑的，而不是排队跑的

常见坑 循环引用

如果A持有`shared_ptr<B>`，B又持有`shared_ptr<A>`，两边的引用计数都无法归零，对象就不会被释放。解决方法通常是把其中一条边改成`weak_ptr`，详见下一节。

### 7.3 std::wead_ptr
概念

`weak_ptr`不拥有对象，只用于观察对象是否仍然存在
 
用法

用之前先`lock()`得到临时`shared_ptr`。

它解决的典型问题
- 断开`shared_ptr`的循环引用，让对象能正常释放
- 在不延长对象生命周期的前提下，选哟时再尝试获取对象

例 用 weak_ptr 断开环
```C++
#include <iostream>
#include <memory>

struct Node {
    int v;
    std::shared_ptr<Node> child;
    std::weak_ptr<Node> parent;
    explicit Node(int x) : v(x) {}
    ~Node() { std::cout << "Node " << v << " destroyed\n"; }
};

int main() {
    auto p = std::make_shared<Node>(1);
    auto c = std::make_shared<Node>(2);
    p->child = c;   // parent 拥有 child
    c->parent = p;  // child 观察 parent, 不形成循环引用

    std::cout << "p.use_count=" << p.use_count() << "\n";
    std::cout << "c.use_count=" << c.use_count() << "\n";

    if (auto pp = c->parent.lock()) {
        std::cout << "parent=" << pp->v << "\n";
    }

    p.reset();

    if (auto pp = c->parent.lock(); !pp) {
        std::cout << "parent expired\n";
    }

    c.reset();
}
```
结果：
```
p.use_count=1
c.use_count=2
parent=1
Node 1 destroyed
parent expired
Node 2 destroyed
```
解读：
- 执行`p->child = c` 共享c对应的内存的有`c`和`p->child`两个`shared_ptr`，所以`c.use_count = 2`
- 执行`c->parent = p` 共享p对应内存的有`<shared_ptr> p `和 `<weak_ptr> c->child `，所以`p.use_count = 1`
## 8.常用STL容器
容器部分建议带着两个问题阅读：最常做的操作是什么，以及是否真的需要有序性。理解这些之后，选对容器往往你写出复杂代码更重要。
### 8.1 vector
概念

连续内存动态数组。默认首选容器

常用初始化
```C++
#include <vector>

int main(){
    std::vector<int> a{1,2,3}; //列表初始化
    std::vector<int> b(5); //5  个0
    std::vector<int> c(5,7); //5个7
    return 0;
}
```
关键接口
- 尾部操作：`push_back`,`emplace_back`,`pop_back`
- 访问元素：`operator[]`,`at`,`front`,`back`
- 大小与容量：`size`,`capacity`,`reserve`,`resize`
- 插入删除：`insert`,`erase`,`clear`
例：
```C++
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v;
    v.reserve(5);

    v.push_back(10);
    v.emplace_back(20);

    std::cout << "size = " << v.size() << ", cap = " << v.capacity() << "\n";
    for (int x : v) std::cout << x << " ";
    std::cout << "\n";
}
```
结果：
```
size = 2, cap = 5
10 20
```

`reverse`和`resize`的区别

`reverse`只影响容量，`resize`真的会改变元素个数。
```C++
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v;

    v.reserve(5);
    std::cout << "after reserve: size=" << v.size() << ", cap=" << v.capacity() << "\n";

    v.resize(5,1);
    std::cout << "after resize:  size = " << v.size() << ", cap = " << v.capacity() << "\n";
    for(int i:v) std::cout << i << " ";
}
```
结果：
```
after reserve: size=0, cap=5
after resize:  size = 5, cap = 5
1 1 1 1 1
```
push_back 和 emplace_back 的区别
- `push_back`把一个对象放进容器，需要先有一个对象
- `emplace_back`在容器里直接构造对象，通常更自然，也可能少一次临时对象。

对`int`这种简单类型，两者没有区别。对需要构造参数的类型，`emplace_back`更常用。

例：emplace_back 的写法更类似构造函数
```C++
#include <iostream>
#include <string>
#include <utility>
#include <vector>

struct Item {
    std::string name;
    int id;
    Item(std::string n, int i) : name(std::move(n)), id(i) {}
};

int main() {
    std::vector<Item> v;
    std::cout << v.size() << "\n";
    v.emplace_back("A", 1);
    std::cout << v.size() << "\n";
    v.push_back(Item{"B", 2});
    std::cout << v.size() << "\n";
}
```
结果
```
0
1
2 
```
删除元素

删除单个位置常用`erase`。按条件删除常用第3.7节的erase-remove写法。
```C++
#include <algorithm>
#include <iostream>
#include <vector>

int main() {
    std::vector<int> v{1, 2, 3, 4, 5, 6};

    v.erase(v.begin() + 1); // 删除第 2 个元素
    v.erase(std::remove_if(v.begin(), v.end(), [](int x) { return x % 2 == 0; }), v.end()); // 删除偶数

    for (int x : v) std::cout << x << " ";
    std::cout << "\n";
}
```
结果：
```
1 3 5
```
在使用迭代器时使用`erase`，会让被删位置之后的迭代器全部失效，因为元素会向前移动填空。
错误示例：
```C++
#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> v = {1, 2, 2, 2, 3};

    // 错误写法：erase 后继续用旧迭代器
    for (auto it = v.begin(); it != v.end(); ++it) {
        if (*it == 2) {  // 删除所有 2
            v.erase(it); //  迭代器直接失效！
        }
    }

    // 你以为会输出：1 3
    // 实际输出：1 2 3 （漏删了！）
    for (int x : v) {
        cout << x << " ";
    }
    return 0;
}
```
结果
```
1 2 3
```
当`vector`扩容时, 它会重新分配一块更大的连续内存并搬家。扩容发生后:
- 之前拿到的元素地址，指针，引用，迭代器都会失效。
- 这也是为什么经常会看到`reserve`的原因

错误示例：
```C++
#include <iostream>
#include <vector>
using namespace std;

int main() {
    vector<int> v;
    v.reserve(3); // 预分配容量3

    // 先插入3个元素，此时容量刚好满
    v.push_back(10);
    v.push_back(20);
    v.push_back(30);

    // 保存扩容前的迭代器、指针、引用
    auto it = v.begin() + 1;  // 指向 20
    int* ptr = &v[2];         // 指向 30
    int& ref = v[0];          // 引用 10

    cout << "扩容前：\n";
    cout << "it: " << *it << "\n";   // 20
    cout << "ptr: " << *ptr << "\n"; // 30
    cout << "ref: " << ref << "\n";  // 10

    //  插入第4个元素，触发扩容（容量从3 → 6或更大）
    v.push_back(40);

    cout << "\n扩容后（注意：以下访问都是未定义行为！）：\n";
    // 这些迭代器/指针/引用已经指向无效内存！
    // 实际运行可能崩溃、输出垃圾值或看似正常（偶然）
    cout << "it: " << *it << "\n";   
    cout << "ptr: " << *ptr << "\n"; 
    cout << "ref: " << ref << "\n";  

    return 0;
}
```
结果：
```
扩容前：
it: 20
ptr: 30
ref: 10

扩容后（注意：以下访问都是未定义行为！）：
it: 616
ptr: 1523515728
ref: 1523542144
```
### 8.2 deque
概念

双端队列，头尾插入删除都高效

什么时候选deque
- 需要频繁`push_back`或`pop_front`
- 想要队列语义，但又希望能随机访问元素

常用接口
- 头尾操作：`push_back`,`push_front`,`pop_front`,`pop_back`
- 随机访问：`operator[]`,`at`
- 大小：`size`,`empty`

和vector的重要区别
- `deque`的存储通常不是一整块连续内存，所以不适合用`date()`交给连续数组的底层接口
- 能更高效地在头部做插入删除

示例：
```C++
#include <deque>
#include <iostream>

int main() {
    std::deque<int> dq;
    dq.push_back(2);
    dq.push_front(1);
    dq.push_back(3);

    dq.pop_front();

    for (int x : dq) std::cout << x << " ";
    std::cout << "\n";
}
```
结果：
```
2 3
```
### 8.3 stack

概念

后进先出容器适配器。只暴露栈语义接口。

它是什么

`std::stack`不是一个真正的底层容器，而是对某个容器的包装。默认底层容器是`std::deque<T>`

这也是为什么`stack`没有迭代器，只是可以做`push`，`pop`，防止你写出依赖内部结构的代码。

常用接口
- `push`,`emplace`
- `top`
- `pop`
- `empty`,`size`

例：
```C++
#include <iostream>
#include <stack>

int main() {
    std::stack<int> st;
    st.push(1);
    st.push(2);

    std::cout << st.top() << "\n";
    st.pop();
    std::cout << st.top() << "\n";
}
```
结果
```
2
1
```

常见坑
- `pop()`不返回被弹出的只，你需要先`top()`再`pop()`
- 空栈上调用`top()`是未定义行为，用前先`empty`。
### 8.4 map
(如果有过python学习经验，可以按照python中的字典来对应理解)

概念

有序键值对。按key排序，查找和插入通常是对数复杂度

什么时候选map
- 需要按key有序遍历
- 需要做范围查询，例如找第一个大于等于某个key的位置
- 数据规模不大，更重视行为稳定与可读性

例：
```C++
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string, int> mp;
    mp["alice"] = 90;
    mp["bob"] = 95;

    if (auto it = mp.find("alice"); it != mp.end()) {
        std::cout << it->first << ": " << it->second << "\n";
    }

    for (const auto& [k, v] : mp) {
        std::cout << k << " " << v << "\n";
    }
}
```
结果
```
alice: 90
alice 90
bob 95
```
注：

`mp[key]`在key不存在时会插入默认值。纯查询尽量用`find`。

查询接口的区别
- `find(key)`只查不改，没找到返回`end()`
- `operator[]`查不到会插入默认值，适合计数或需要确保存在的场景
- `at(key)`查不到会抛异常，适合你认为key必须存在的场景

插入接口对比

`insert`与`emplace`都用于插入元素，并会告诉你插入是否成功。
`try_emplace`和`insert_or_assign`是C++17更常见的写法
- `try_emplace(key,args...)`只有key不存在时才构造value，避免不必要构造
- `insert_or_assign(key,value)`key不存在就插入，存在就覆盖
- `insert`,`emplace`,`try_emplace`的返回值都是pair([迭代器，int])
```C++
#include <iostream>
#include <map>
#include <string>

int main() {
    std::map<std::string, int> mp;

    auto [it1, ok1] = mp.insert({"alice", 90});
    std::cout << ok1 << " " << it1->second << "\n";

    auto [it2, ok2] = mp.try_emplace("alice", 95); // 已存在, 不会覆盖
    std::cout << ok2 << " " << it2->second << "\n";

    mp.insert_or_assign("alice", 100); // 覆盖
    std::cout << mp["alice"] << "\n";
}
```
结果
```
1 90
0 90
100
```
find 与 迭代器
- `mp.find(key)`返回一个迭代器
- 如果没找到，返回`mp.end()`。
- 找到时，迭代器指向一个键值对，可以用`it->first`访问`key`，用`it->second`访问 `value`。

见第2.4节：`if(auto it = mp.find("alice);it != mp.end()){ ... }`它的好处是把`it`的作用域限制在if语句内部。
### unordered_map
概念

哈希表映射。平均查找接近常数时间。

什么时候选unordered_map
- 只关心查找和插入速度，不关心遍历顺序
- key的比较和排序很麻烦，但能方便地做哈希

常用接口
- `find`,`operator[]`,`at`的语义与`map`类似
- `reserve(n)`提前为n个元素预留桶，减少rehash
- `rehash(n)`直接设置桶数量 

例：
```C++
#include <iostream>
#include <string>
#include <unordered_map>

int main() {
    std::unordered_map<std::string, int> freq;
    for (std::string s : {"a", "b", "a", "c", "a"}) {
        ++freq[s];
    }

    std::cout << "a => " << freq["a"] << "\n";
}
```
结果：
```
a => 3
```
注：不要依赖遍历顺序。它不保证稳定

reserve的意义

`unordered_map`会在元素增多时自动扩容并rehash。rehash会搬动元素，这会让迭代器失效，也会带来一段时间的性能波动。当你知道大概规模时，提前reserve往往更稳。
```C++
#include <iostream>
#include <string>
#include <unordered_map>

int main() {
    std::unordered_map<std::string, int> freq;
    freq.reserve(1000);
    freq["a"] = 1;
    std::cout << freq["a"] << "\n";
}
```
结果
```
1
```
自定义key

如果key不是标准库自带类型，你需要提供：
- 哈希函数
- 相等比较

否则编译器不知道怎么把key放进哈希表 
### 8.6 set

**概念**

有序去重集合

**什么时候选set**
- 只关心元素是都存在，不需要value
- 需要有序遍历或范围查询

**常用接口**
- `insert`插入，返回插入是否成功
- `erase`删除
- `find`查找
- `count`判断是否存在，结果是0或1
- `lower_bound`找第一个不小于某个值的**位置**

例：
```C++
#include <iostream>

int main() {
    std::set<int> s{3, 1, 2, 2};

    std::cout << "\n";

    auto it = s.lower_bound(3);

}
```
结果：
```

```

### 8.7 priority_queue
**概念**

优先队列。默认是大顶堆

**常用接口**
- `push`,`emplace`
- `top`取出当前最优先的元素
- `pop`弹出当前最优先的元素
- `empty`，`size`
  
**小顶堆写法**
`priority_quene<T,vector<T>,greater<T>>`

例：
```C++
#include <functional>
#include <iostream>
#include <queue>
#include <vector>

int main() {
    std::priority_queue<int> max_heap;
    max_heap.push(3);
    max_heap.push(1);
    max_heap.push(5);
    std::cout << max_heap.top() << "\n";

    std::priority_queue<int, std::vector<int>, std::greater<int>> min_heap;
    min_heap.push(3);
    min_heap.push(1);
    min_heap.push(5);
    std::cout << min_heap.top() << "\n";
}
```
结果：
```
5
1
```
**常见坑**

堆只保证top是最值，不保证遍历有序

**自定义比较规则**

如果你存的是自定义结构体，你需要提供比较器来决定谁的优先级高更高。比较器的含义可以理解成排序规则，但不要把它当成稳定排序
```C++
#include <iostream>
#include <queue>
#include <vector>

struct Job {
    int priority;
    int id;
};

struct JobLess {
    bool operator()(const Job& a, const Job& b) const {
        return a.priority < b.priority; // priority 大的在 top
    }
};

int main() {
    std::priority_queue<Job, std::vector<Job>, JobLess> pq;
    pq.push(Job{3, 1});
    pq.push(Job{10, 2});
    pq.push(Job{5, 3});

    std::cout << pq.top().priority << " id = " << pq.top().id << "\n";
}
```
输出：
```
10 id = 2
```

速查表
|容器         |随机访问        |头部插入删除   |尾部插入删除      |按键查找        |
|-------------|---------------|---------------|-----------------|---------------|
|vector   |O(1)|O(n)|均摊O(1)|不适用|
|deque|O(1)|O(1)|O(1)|不适用|

## 9.并发与异步
### 9.1 竞态条件与数据竞争
很多人第一次接触并发时，以为问题时线程跑的快慢不一样，爱是本质时线程会在几乎任何语句之间被操作系统打断，于是同一段代码在不同的交错顺序下回产生不同的交错顺序下会产生不同的中间状态。只要你的程序依赖某个特定交错顺序才正确，你就遇到了竞态条件

#### 这里需要区分两个经常被混用的此：
- 竞态条件指结果依赖时间与交错顺序的逻辑错误，它描述的时你的算法假设与现实调度之间的冲突 
- 数据竞争是C++里一个更严格的术语，它指多个线程在没有建立同步关系的前提下访问同一块内存，且至少有一次写，一旦发生就是未定义行为。

数据竞争一定是竞态条件，竞态条件不一定是数据竞争

`std::thread`：
- `<thread>`头文件里提供`std::thread`与`std::this_thread`
- `std::thread t(f)`会立刻启动一个新线程来执行`f`
- `t.join()`会让当前线程等待`t`对应的线程结束，线程结束后会回收底层资源
- `std::thread`对象析构时如果仍然joinable,也就是它孩子啊管理一个线程，程序会`std::terminate`，所以每个线程都必须被`join`或`detach`
 
#### 例：
```C++
#include <iostream>
#include <thread>

int main() {
    int counter = 0;

    auto worker = [&] {
        for (int i = 0; i < 100000; ++i) {
            ++counter; // 数据竞争, 未定义行为
        }
    };

    std::thread t1(worker);
    std::thread t2(worker);
    t1.join();
    t2.join();

    std::cout << "counter=" << counter << "\n";
}
```
#### 结果
```
count为100000-200000之间的一个数
```
这段代码的问题在于`++counter`不是一个不可分割的动作。它在机器层面会被拆成三步：读出旧值，加一，写回新值。两个线程如果刚好都读到了同一个旧值，最终就会把同一个新值写回去，其中一次加一会被覆盖掉，于是你看到的结果通常小于`200000`，且完全不可预测。

如果在并发代码例看到了这样的模式，你应该立刻停下来问自己，这个共享变量的读写是否已经通过互斥锁、原子操作、或者条件变量建立的同步关系，否则他就处在未定义行为的范围里。

#### 例 没有数据竞争也可能有竞态条件

我们用`std::atomic`把数据竞争消掉，但逻辑依然可能错误。

这里用到的`std::atomic<bool>`来自`<atomic>`。它表示一个原子变量，你可以用`load()`原子读取，用`store(v)`原子写入，从而保证对这个布尔值本身的并发读写不会产生数据竞争。注意这并不意味着一段多步逻辑会自动变成不可分割的整体，`if`的判断与`init()`的执行仍然可能被其它线程插进来。

#### 例1：
```C++
#include <atomic>
#include <iostream>
#include <thread>

void init() {
    std::cout << "do init\n";
}

int main() {
    std::atomic<bool> initialized{false};

    auto worker = [&] {
        if (!initialized.load()) {          // 这里没有数据竞争
            init();                         // 但 init 可能被执行多次
            initialized.store(true);
        }
    };

    std::thread t1(worker);
    std::thread t2(worker);
    t1.join();
    t2.join();
}
```
#### 常见输出：
```
do init
do init
```
这段代码的问题在于判断与执行之间缺少约束，两个线程都可能先看到`initialized`为`false`，然后同时进入`init()`。也就是说`std::atomic`解决的是一个内存位置的并发读写安全，它并不自动帮你把多个步骤合并成一个不可分割的整体。这个场景的正确写法通常是`std::call_once`。或者加锁保护初始化过程，他们表达的是更强的语义，也更贴近你的真实需求。

#### 例 用`std::call_once`做一次初始化

`std::call_once`与`std::once_flag`都在`<mutex>`里：

- `std::once_flag`是一次性初始化的状态对象，它不可拷贝，也不能手动重置，你需要把同一个`flag`共享给所有线程。
- `std::call_once(flag,f,args...)`会保证在所有并发调用中，`f(args...)`只会执行一次，其他调用会等待那一次完成后再返回
- 如果那一次初始化抛异常，`flag`不会被职位完成状态，后续调用依然会再次尝试执行初始化。
```C++
#include <iostream>
#include <mutex>
#include <thread>

void init() {
    std::cout << "do init\n";
}

int main() {
    std::once_flag flag;

    auto worker = [&] {
        std::call_once(flag, init);
    };

    std::thread t1(worker);
    std::thread t2(worker);
    t1.join();
    t2.join();
}
```
结果
```
do init
```
`call_once`的语义比一个原子布尔值加`if`更强：它保证`init`在整个进程里指会执行一次，其他线程如果同时跑到这里，会等待那一次初始化完成后再继续向下执行。你不需要自己写等待与唤醒,也不需要自己处理初始化过程里抛异常时标志位如何恢复，这些细节都有标准库完成。
### 9.2 线程是什么，以及`std::thread`的使用方式
在操作系统层面，线程时进程内的执行单元。它和同一进程的其他线程共享地址空间，因而可以直接读写同一份全局变量、堆变量与静态变量；与此同时，每个线程又有自己的寄存器上下文与独立的栈，这也是为什么不同线程里局部变量互不影响。操作系统调度器会在多个线程之间切换执行权，切换点通常不由你的源代码决定，所以你必须假设任何一行代码都可能与其他线程交错。

在C++里，你用`std::thread`来启动一个线程。理解它的关键不是构造函数的参数列表，而是生命周期管理：
- 线程启动后会并发执行，你需要决定什么时候等待它结束
- `std::thread`对象析构时如果仍然是joinable,也就是`joinable()`返回true，程序会直接`std::terminate`，这是为了避免你无意中把一个还在跑的线程丢进后台

`std::terminate`是一种直接终止程序的机制，一旦触发通常就表现为进程异常退出，所以并发代码里不要把`join`当成可选收尾，而要把它当成线程资源管理的一部分（**一定要写**）。

#### 从类型角度看，`std::thread`还有两个常见细节：
- 它是move-only类型，不能拷贝。你如果要管理多个线程，常见做法是放进`std::vector<std::thread>`并用`emplace_back`直接创建
- 线程函数的参数会先被拷贝进内部存储，然后在线程开始时再传给线程函数，这也是为什么传引用需要显式写`std::ref`

#### 还有几个常用接口在调试与排错时很实用：
- `joinable()`用来判断当前线程对象是否还需要join或detach
- `get_id()`与`std::this_thread::get_id()`用来打印线程标识，帮你把日志和具体线程对应起来。
- `std::thread::hardware_concurrency()`近似给出硬件线程数，常用来估计并行度的上限

另外，你会在本章多处看到`std::this_thread::sleep_for(...)`。它会让当前线程睡眠一段时间，常用来模拟耗时操作或制造更明显的交错，便于观察并发程序的行为。它的参数是一个时间长度类型，来自`<chrono>`，常见写法是`std::chrono::miliseconds(200)`

#### 例 创建线程并等待结束
```C++
#include <iostream>
#include <thread>

int main() {
    std::thread t([] {
        std::cout << "hello from worker\n";
    });

    t.join(); // 等线程结束
}
```
#### 结果：
```
hello from worker
```
join() 这一步不仅是等结果, 更重要的是把生命周期关系写进了代码: 主线程在这里明确表示, 我不会在工作线程结束之前退出作用域, 因而被工作线程使用到的局部变量也就不会提前销毁。

若不是很理解，可以修改9.1中的例1程序如下：
```C++
#include <iostream>
#include <thread>

int main() {
    int counter = 0;
    auto worker = [&] {
        for (int i = 0; i < 100000; ++i) {
            ++counter; // 数据竞争, 未定义行为
        }
    };
    std::thread t1(worker);
    t1.join();
    
    std::thread t2(worker);
    t2.join();

    std::cout << "counter=" << counter << "\n";
}
```
#### 结果：
```
counter=200000
```
#### 线程传参 复制与引用
`std::thread`默认会把参数拷贝一份给线程函数，这在很多情况下是安全的，但是如果你想传引用，就需要显式写`std::ref`。`std::ref(x)`会返回一个引用包装器，让`std::thread`在内部保存引用而不是拷贝值，线程函数最终拿到的能然是`int&`。这不是语法技巧，而是为了逼你意识到引用参数的生命周期风险。
#### 例：
```C++
#include <functional>
#include <iostream>
#include <thread>

void add_one(int& x) { x += 1; }

int main() {
    int x = 0;
    std::thread t(add_one, std::ref(x));
    t.join();
    std::cout << x << "\n";
}
```
#### 结果：
```
1
```

#### `detach()`的风险
`detach()`会让线程在后台继续运行，你失去了对它何时结束的控制。后台线程一旦还在访问某个对象，而这个对象已经被销毁，程序就会立刻进入未定义行为。除非你能用清晰的所有权模型证明对象一定活得足够久，否则初学阶段应当尽量避免`detach()`。

如果你希望写出更不容易忘记`join`的代码，C++20提供了`std::jthread`，它在析构时会自动join并且配合`stop_token`支持协作式停止；这部分在第十章展开。 

#### 线程函数里的异常
如果异常从线程函数直接抛出到线程边界之外，标准库会调用`std::terminate`。因此线程函数里要么捕获异常并自行处理，要么把异常信息通过`std::promise`或者其他通道传回主线程。`std::promise`是`<future>`里用于跨线程传递结果或异常的工具，不要指望主线程能自动catch到工作线程的异常。
